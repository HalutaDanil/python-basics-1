## 🇬🇧 English

Ten algorithmic tasks covering core Python concepts. Each task isolates a specific idea — from basic math to dynamic programming — without relying on external libraries.

### What was done

| Task | What & Why |
|------|-----------|
| Scalar Product | Computed the dot product of two 3D vectors. Practice with arithmetic and `stdin/stdout`. |
| Palindrome Check | Determined if an integer is a palindrome **without converting to string**. Used modulo and division to reverse digits arithmetically. |
| Figures in Matrix | Counted "squares" and "circles" in a binary matrix by analyzing connected components of 1s. |
| Pascal's Triangle | Generated N rows of Pascal's triangle iteratively. Practice with nested loops and list construction. |
| Custom float() | Parsed a string into a float manually — handling sign, decimal point, and exponent. Deepened understanding of how Python parses numbers. |
| Movie Merge | Merged two sorted JSON movie lists into one sorted list. Practice with JSON I/O and merge algorithm. |
| Robot Coin Collector | Solved a classic DP problem: a robot moves only right or down on a grid, collecting coins. Used `dp[i][j] = max(dp[i-1][j], dp[i][j-1]) + coins[i][j]`. |
| Unique Numbers | Counted distinct integers from input using a set. Simple but fundamental data structure practice. |
| Polynomial Derivative | Computed the derivative of a polynomial at a given point. Applied basic calculus in code. |
| Machine Selection | Selected two machines with matching year and minimal total cost that satisfy a runtime requirement. Combined filtering, sorting, and constraint satisfaction. |

### Key takeaways
- Python's standard library is powerful enough for most algorithmic challenges.
- **Dynamic programming** and **manual parsing** are skills that transfer directly to technical interviews.
- Avoiding string conversion for numeric tasks deepens understanding of how numbers work at the bit/arithmetical level.

## 🇷🇺 Русский

Десять алгоритмических задач, покрывающих ключевые концепции Python. Каждая задача изолирует конкретную идею — от базовой математики до динамического программирования — без использования внешних библиотек.

### Что было сделано

| Задача | Что и зачем |
|--------|-------------|
| Скалярное произведение | Вычисление скалярного произведения двух векторов в 3D. Практика арифметики и `stdin/stdout`. |
| Палиндром | Проверка числа на палиндром **без преобразования в строку**. Использованы `%` и `//` для арифметического разворота цифр. |
| Фигуры в матрице | Подсчёт "квадратов" и "кругов" в бинарной матрице через анализ связных областей единиц. |
| Треугольник Паскаля | Генерация N строк треугольника Паскаля итеративно. Практика вложенных циклов и построения списков. |
| Свой float() | Ручной парсинг строки в float — обработка знака, десятичной точки и экспоненты. Углублённое понимание парсинга чисел в Python. |
| Слияние фильмов | Слияние двух отсортированных JSON-списков фильмов в один отсортированный. Практика с JSON I/O и алгоритмом слияния. |
| Робот-сборщик монет | Классическая задача на ДП: робот движется только вправо и вниз по полю, собирая монеты. Формула: `dp[i][j] = max(dp[i-1][j], dp[i][j-1]) + coins[i][j]`. |
| Уникальные числа | Подсчёт различных целых чисел через множество. Простая, но фундаментальная практика со структурами данных. |
| Производная многочлена | Вычисление производной многочлена в заданной точке. Применение базового анализа в коде. |
| Выбор машин | Подбор двух машин с совпадающим годом и минимальной суммарной стоимостью под требование по времени работы. Фильтрация, сортировка и удовлетворение ограничений. |

### Ключевые выводы
- Стандартной библиотеки Python достаточно для большинства алгоритмических задач.
- **Динамическое программирование** и **ручной парсинг** — навыки, напрямую применимые на технических собеседованиях.
- Отказ от строкового преобразования для числовых задач углубляет понимание работы чисел на арифметическом уровне.
